# Estrategia de Seguimiento y Soporte Post Proyecto
## Sistema de Control de Accesos

### 1. Introducción

Una vez finalizado el desarrollo e implementación del sistema, es fundamental establecer un proceso de seguimiento y soporte que garantice su correcto funcionamiento, evolución y mantenimiento.

El soporte post proyecto busca asegurar la estabilidad del sistema, atender incidentes y mejorar continuamente la solución.

---

# 2. Objetivos del Soporte

Los principales objetivos del soporte post implementación son:

- Garantizar la disponibilidad del sistema.
- Resolver errores detectados por los usuarios.
- Atender solicitudes de mejora.
- Mantener actualizado el sistema.
- Asegurar la continuidad operativa.

---

# 3. Tipos de Soporte

## 3.1 Soporte Correctivo

Se encarga de solucionar errores o fallos que aparezcan durante la operación del sistema.

Ejemplos:

- Problemas de conexión con lectores RFID
- Errores en el registro de accesos
- Fallos en la generación de reportes

---

## 3.2 Soporte Preventivo

Busca evitar fallos antes de que ocurran.

Actividades:

- Actualización de software
- Verificación del servidor
- Revisión de base de datos
- Monitoreo de rendimiento

---

## 3.3 Soporte Evolutivo

Incluye mejoras o nuevas funcionalidades solicitadas por la organización.

Ejemplos:

- Integración con sistemas de nómina
- Control biométrico
- Integración con cámaras de seguridad
- Aplicación móvil de monitoreo

---

# 4. Niveles de Soporte

### Nivel 1 – Soporte Básico

Atención inicial a los usuarios.

Responsabilidades:

- Resolución de problemas simples
- Restablecimiento de contraseñas
- Soporte en uso del sistema

---

### Nivel 2 – Soporte Técnico

Atiende problemas técnicos más complejos.

Responsabilidades:

- Diagnóstico de errores
- Revisión de base de datos
- Configuración del sistema

---

### Nivel 3 – Soporte de Desarrollo

Intervención directa del equipo desarrollador.

Responsabilidades:

- Corrección de bugs
- Optimización del sistema
- Implementación de mejoras

---

# 5. Gestión de Incidentes

El proceso de gestión de incidentes incluye:

1. Registro del incidente
2. Clasificación del problema
3. Asignación al responsable
4. Diagnóstico
5. Solución
6. Validación con el usuario
7. Cierre del incidente

---

# 6. Monitoreo del Sistema

Para garantizar la estabilidad del sistema se recomienda monitorear:

- Uso de CPU del servidor
- Uso de memoria
- Conexiones de red
- Disponibilidad de la base de datos
- Eventos del sistema

---

# 7. Mantenimiento del Sistema

Se recomienda realizar mantenimientos periódicos como:

- Limpieza de base de datos
- Optimización de consultas
- Actualización de dependencias
- Copias de seguridad

---

# 8. Gestión de Backups

El sistema debe contar con un proceso automático de copias de seguridad que incluya:

- Backup diario de base de datos
- Backup semanal del sistema
- Almacenamiento seguro de backups

---

# 9. Canal de Atención

Los usuarios podrán reportar problemas a través de:

- Correo electrónico
- Sistema de tickets
- Área de soporte TI

---

# 10. Mejora Continua

Se realizarán revisiones periódicas del sistema para identificar oportunidades de mejora.

Estas revisiones incluirán:

- Análisis de uso
- Retroalimentación de usuarios
- Evaluación del rendimiento
